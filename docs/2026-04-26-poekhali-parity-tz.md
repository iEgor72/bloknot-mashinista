# ТЗ: перенос функционала APK в Блокнот машиниста

Дата: 2026-04-26

Цель: вести перенос не по ощущениям, а по проверяемой матрице. Каждый пункт из оригинального приложения должен иметь один из статусов: `ГОТОВО`, `ЧАСТИЧНО`, `НЕТ`, `НУЖНА ПРОВЕРКА APK`. Заглушки, демо-режимы и отдельный ввод данных, которые уже есть в карточке смены, не считаются готовым функционалом.

## Функционал оригинального приложения

### 1. Карточки поездок и смен

Статус источника: подтверждено по help-страницам APK и предыдущему разбору.

- Добавление поездки/смены через отдельную форму.
- Ввод данных по поезду: номер, вес, оси, условная длина, род/маршрут.
- Ввод данных по локомотиву: серия, номер, параметры для расчета.
- Расчет и сохранение маршрута/поездки.
- Отображение сохраненных маршрутов со сведениями о рабочем времени и экономии/пережоге.
- Резервное сохранение данных поездок через настройки.

### 2. Расчеты топлива и технической скорости

Статус источника: подтверждено по help-страницам APK, формулы требуют отдельной полной сверки.

- Расчет нормы топлива по участку обслуживания.
- Учет веса, осей и нагрузки на ось.
- Учет серии тепловоза, включая разные нормы для отдельных исполнений.
- Учет толкача: станции прицепки/отцепки, вес на головной локомотив, доля веса на толкач.
- Учет троганий с места, троганий с толкачом.
- Учет горячего простоя и стоянок.
- Ручной расчет для неполного прохождения участка: длина участка, удельный расход, нормы на трогание и простой.
- Расчет фактической технической скорости с автозаполнением данных из расчета простоя.

### 3. Расчет заработной платы

Статус источника: частично подтверждено предыдущими требованиями и текущими правками, нужна полная сверка APK.

- Расчет по месячной норме и фактически отработанным часам.
- Тарифная ставка.
- Ночные/вечерние/праздничные/выходные доплаты.
- Районные, зональные, БАМ и прочие процентные надбавки.
- Командировочные.
- Итоговые начисления за месяц.

### 4. Режим `Поехали`

Статус источника: подтверждено по APK, EMap и поведению оригинала.

- Отдельная кнопка/режим `Поехали`.
- Запрос GPS и показ текущего состояния GPS.
- Автоматическое определение положения на карте по координатам.
- Автоматическое определение участка/сектора и текущей координаты.
- Показ текущей скорости, точности, времени обновления.
- Работа с электронными картами EMap:
  - `data.xml` - геометрия/координаты маршрута;
  - `profile.xml` - профиль пути и уклоны;
  - `speed.xml` - скоростные ограничения;
  - объектные XML - станции, светофоры, скоростные знаки и прочие объекты.
- Загрузка/импорт карт из источника APK и хранение в приватном хранилище приложения.
- Отрисовка профиля пути, а не условной прямой линии.
- Поезд находится на линии профиля и движется по профилю.
- Пикеты/километры, станции, светофоры, скорости и уклоны привязаны к профилю пути.
- Переключение направления/четности и пути.
- Работа с длиной состава.
- Ввод предупреждений.
- Отображение активных ограничений/предупреждений в режиме движения.
- Работа без ручного выбора участка в штатном сценарии: приложение само понимает, где поезд.

### 5. Справочники и помощь

Статус источника: подтверждено по assets APK.

- Справка по расчетам.
- Справочник номеров поездов.
- Табличные материалы по нормам топлива.
- Локальные изображения/таблицы в составе APK.

## Что уже есть в нашем приложении

### 1. Карточки смен

Статус: `ГОТОВО`, но нужно использовать эти данные во всех связанных режимах.

- Добавление смены.
- Поля локомотива: серия и номер.
- Поля поезда: номер, вес, оси, условная длина.
- Маршрут: тип, откуда, куда.
- Сохранение смен и отображение в календаре/списке.
- `Поехали` уже читает активную, выбранную вручную или последнюю смену и берет оттуда поезд, локомотив, условную длину, вес, оси и маршрут.
- `Поехали` записывает итог поездки обратно в эту же смену: статус записи, карта, направление, путь, старт/финиш, время, время в движении/стоянке, дистанцию, среднюю/техническую/максимальную скорость, счетчик ПР, активное/следующее ограничение, ближайший светофор/станцию впереди, прогресс по маршруту смены, превышения ограничения и координаты начала/конца.
- Активная или поставленная на паузу запись `Поехали` удерживает привязку к своей смене после перезагрузки, а длительность записи, направление и путь восстанавливаются из сохраненной записи.
- Кнопка `Запись` перед созданием поездки сначала готовит рабочий контекст смены: подтягивает карту, маршрут, направление, путь и стартовую точку, чтобы первый run не создавался пустым техническим состоянием. Если нет выбранной/активной смены, GPS-проекции или привязанного маршрута на ЭК, запись не стартует и показывает причину в `ПР`/статусе.
- В панели `ПР` можно явно выбрать смену для `Поехали`; активная запись перепривязывается к выбранной карточке и дальше пишет итоговые поля именно в нее.
- Временный UX 26.04.2026: пункт `Поехали` убран из нижнего меню, чтобы не ломать баланс основной навигации; вход оставлен через подробности смены и меню действий смены. При прямом входе из смены режим фиксирует выбранную карточку, синхронизирует предупреждения и до первой рабочей отрисовки запускает подбор ЭК/стартовой точки по маршруту карточки.
- Из подробностей смены можно открыть `Поехали` сразу на этой смене: режим выбирает карточку, подбирает карту по маршруту и берет поезд/локомотив/длину без отдельного ввода.
- Из меню действий смены можно открыть `Поехали` напрямую, без промежуточного экрана подробностей.
- Редактирование смены сохраняет поля `poekhali_*`, карточка и экран подробностей смены показывают данные поездки.

### 2. Режим `Поехали`

Статус: `ЧАСТИЧНО`, базовый рабочий слой есть, parity еще не закрыт.

Готово:

- Есть отдельный экран/режим `Поехали`; временно вход идет не из нижней навигации, а из карточки смены и меню действий смены.
- Загружается восстановленная EMap `Комсомольск ТЧЭ-9`.
- Поддержаны 20 участков карты.
- Парсятся `data.xml`, `profile.xml`, `speed.xml`, `1.xml`, `1n.xml`, `2.xml`, `2n.xml`.
- GPS проецируется на маршрут и дает участок/координату.
- Есть отображение км/пк.
- Есть переключатели `КМ/K`, чет/нечет, путь.
- Есть выбор карты и выбор участка для просмотра без GPS.
- Есть no-GPS preview с перетаскиванием по участкам.
- Профиль пути, станции, светофоры, скорости, уклоны, км/пк уже рисуются на canvas.
- Поезд рисуется на профиле, длина поезда считается из условной длины карточки смены; если условной длины нет, режим больше не подставляет демо-состав 1045 м, а явно использует расчет по осям или локомотив с предупреждением.
- Нижнее меню приложения остается в режиме `Поехали`, но сам пункт `Поехали` временно убран из нижней навигации; экран открывается из смены.
- Есть панель `ПР`: карточка смены плюс ввод предупреждений.
- Предупреждения привязаны к карте и смене, сохраняются локально/на сервере, рисуются на профиле как `ПР` и записывают компактный паспорт правил в связанную смену.
- Активное ограничение выбирается единым правилом `ПР > ДОК > РК > ЭК`, показывается в HUD/нижней панели, сохраняется в журнал поездки и записывается в связанную смену.
- Следующее ограничение впереди выбирается из тех же источников по направлению движения, показывается в HUD, отдельной меткой `Далее` на профиле для ДОК/РК/ЭК и записывается в журнал/смену с расстоянием до входа.
- Ближайший светофор и ближайшая станция по направлению движения определяются по объектам текущего пути/участка, показываются в журнале `Поехали` и записываются в связанную смену с расстоянием и координатой.
- Прогресс по маршруту смены считается от сопоставленных станций `откуда/куда`: пройдено, осталось, процент и полная длина маршрута показываются в панели маршрута/журнале, пишутся в связанную смену и видны на рабочем экране `Поехали` как прогресс-рейка маршрута с меткой ближайшего старта/финиша на профиле.
- ETA до следующего ограничения, светофора, станции и финиша маршрута считается только по текущей фактической скорости; при остановке или отсутствии скорости показывается пустое значение, чтобы не давать ложный прогноз.
- `Цель впереди` выбирается автоматически из ближайших важных событий по ходу: конец действующего ограничения, следующее ПР/ограничение, светофор, станция, старт или финиш маршрута. Цель показывается в рабочем HUD, журнале `Поехали` и записывается в связанную смену с расстоянием, координатой и ETA.
- Если фактическая скорость выше активного ограничения, скорость подсвечивается как превышение, а максимальное превышение, время и путь превышения пишутся в журнал и смену.
- Рабочий экран `Поехали` теперь дает runtime-предупреждения: при превышении активного ограничения и при приближении к выбранной `Цели впереди` за 1000/300 м верхняя панель меняет тон и текст, а устройство получает тактильный сигнал через доступный `BM_HAPTICS`. Сработавшие оповещения пишутся в run и связанную смену: счетчик, последнее оповещение, тип, дистанция и время видны в панели `ПР`, журнале и подробностях смены.
- SW кеширует карты и обновляет кодовые ассеты network-first.
- `Контроль prod` разделен на готовность самого режима и готовность текущей поездки: отсутствие выбранной смены/GPS больше не маскируется под недоделку функционала, а показывается отдельным эксплуатационным блокером запуска. Пакет скачанных ЭК проверяется по рабочему слою `ЭК+GPS+РК+ДОК`, чтобы сырой XML не блокировал production-готовность там, где профиль/скорости закрыты режимными картами, GPS-слоем или актуальными документами.
- Панель `ПР` приведена к пользовательскому виду: в обычном режиме видны только вкладки `Поездка`, `ПР`, `Карта`; технический prod-аудит, диагностика, резерв, подробная готовность XML/слоев и служебные списки доступны только через скрытый debug-режим `?poekhaliDebug=1`, чтобы машинист не видел техническую простыню во время работы.

Частично:

- Автоопределение позиции есть: текущая ЭК выбирается по GPS среди локально установленных карт, а до GPS при входе в режим подбирается по маршруту карточки смены. Remote-карты из `ek_files` сняты с production-blocker scope; недостающие участки закрываются локальной ЭК, GPS-дорисовкой, РК и ДОК.
- Поезд на профиле рисуется как состав по длине из карточки смены: тело идет по фактической линии профиля от хвоста до головы, есть локомотивная голова, деления состава и колеса. Без GPS в просмотре ЭК показывается полупрозрачный состав-ориентир, при GPS он становится рабочим live-составом. Остается сверить визуальную детализацию с APK на реальных поездках.
- Длина состава связана с карточкой смены: условная длина имеет приоритет, при ее отсутствии возможен расчет по осям, а резерв/одиночный локомотив/незаполненная поездка показываются без выдуманного грузового состава. Не закрыта отдельная доменная модель для сплотки, пассажирского состава и толкача.
- Предупреждения можно добавить, редактировать, отключить, удалить и ограничить сроком действия; в панели показывается статус `действует/впереди/пройдено/отключено/истекло`, а ближайшее ПР по направлению движения выводится в рабочем HUD и подсвечивается на профиле. ПР синхронизируются с сервером через `/api/poekhali-warnings`, локальные и серверные изменения сливаются по `updatedAt/deletedAt`, удаление идет мягкой меткой, чтобы старые серверные записи не воскресали; активные/общие ПР и краткий список теперь видны в карточке и подробностях связанной смены. Пакетный ввод ПР и импорт локального `.txt/.csv/.json` файла сохранены, но скрыты в debug-режиме, чтобы рабочая вкладка ПР оставалась простой. Остаются не закрыты: распознавание ПР напрямую из PDF/сканов и расширенные уведомления.
- Скорости и предупреждения рисуются; базовый контроль активного ограничения готов: предупреждение перекрывает документ, документ перекрывает режимную карту и ЭК, а при равном источнике выбирается более строгая скорость. Конфликт ЭК/ДОК теперь открывает карточку с исходной строкой из индекса документа, страницей PDF, кнопкой открытия источника и локальной сверкой `сверено OK/ошибка парсера`; статус сверки входит в `Контроль prod`.
- Часть участков честно помечается как route-only, если в EMap нет профиля. Это не баг, но для 100% готовности нужна карта/профиль по всем участкам, где они должны быть.
- Направление, путь и стартовый участок привязаны к маршруту карточки смены: `Поехали` ищет станции `откуда/куда` во всех скачанных ЭК, может переключить карту маршрута, выставить `ЧЕТ/НЕЧЕТ`, выбрать `П1/П2` по объектному XML станции и открыть стартовую станцию; при старте записи эти данные повторно применяются перед записью run, если режим еще грузил карту. Старт без GPS разрешен только при надежной привязке маршрута смены к профилю, чтобы не плодить пустые рабочие записи. Если карта не дает надежного объектного файла, путь остается ручным переключателем.
- Управление источниками карты из UI частично готово: в обычной вкладке `Карта` оставлены только понятная готовность маршрута, покрытие, сверка и смена карты. Подробная проверка пакета карт, диагностика, резерв, prod-аудит, ЭК/РК/ДОК-списки и Cache/API-сведения спрятаны в скрытый debug-режим, чтобы не перегружать рабочий сценарий. Не закрыты ручное удаление/обновление локально установленных карт.

Осталось/не закрыто:

- Встроенное APK-like скачивание карт из `ek_files` исключено из текущего scope, потому что рабочего источника нет; режим должен развиваться через локальные ЭК, GPS-дорисовку, РК и ДОК.
- Backend-синхронизация предупреждений между устройствами добавлена; пакетный ручной ввод и импорт локального `.txt/.csv/.json` файла в панели `ПР` добавлены. Не закрыты распознавание ПР напрямую из PDF/сканов и расширенные уведомления.
- Журнал/история поездок из режима `Поехали` добавлен как технический слой синхронизации, а пользовательский итог пишется в связанную смену: старт/пауза идут от кнопки `Запись`, GPS обновляет последнюю точку, дистанцию, длительность, время движения/стоянки, среднюю/техническую/максимальную скорость, счетчик ПР, текущее активное/следующее ограничение, ближайший светофор/станцию, прогресс маршрута и превышения. Не закрыты расширенные отчеты по поездке и нормативный расчет топлива из APK.

## Definition of Done для `Поехали`

Функция считается готовой только если:

- режим запускается отдельной кнопкой `Поехали` из карточки/меню смены; нижний пункт навигации временно не используется;
- без GPS показывает реальную карту/профиль для просмотра, а не демо;
- с GPS сам определяет карту, участок, км/пк и показывает поезд на профиле;
- данные состава берутся из карточки смены;
- все сущности EMap отрисованы на профиле без критичных наложений;
- предупреждения вводятся, сохраняются, отображаются и влияют на активное ограничение;
- проверены все участки установленной карты;
- проверены мобильные ширины 320, 360, 390;
- нет JS-ошибок, битых ассетов, пустого canvas и скрытых заглушек.

Текущий контроль DoD вынесен в `ПР -> Контроль prod`: автоматические пункты показывают production-блокеры по данным/картам/синхронизации, а блокеры конкретной поездки (`нет смены`, `нет маршрута`, `нет живой GPS-проекции`) вынесены в отдельный блок `Готовность текущей поездки`. Ручные пункты фиксируют приемку мобильных ширин, реальной GPS-поездки, визуальной сверки APK и наложений на профиле.
